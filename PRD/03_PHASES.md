## Phase 1: MVP

### 목표
광고 대본 또는 USP를 입력하면 가상 소비자들이 반응하고, 분석 보고서를 받을 수 있는 상태.

### 기능
- [ ] 시드 자료 업로드 (광고 대본, USP 텍스트 붙여넣기 + PDF 업로드)
- [ ] 시뮬레이션 유형 선택 (광고 반응 / USP 테스트)
- [ ] 타겟 페르소나 설정 (프리셋 선택 + 커스텀 설정)
- [ ] MiroFish 엔진 연동 시뮬레이션 실행 (진행률 표시)
- [ ] 텍스트 분석 보고서 생성 (긍정/부정 비율, 핵심 인사이트, 우려사항, 추천)
- [ ] 에이전트 대화 로그 열람 (개별 가상 소비자 반응 확인)
- [ ] 프로젝트 목록 관리 (생성, 조회, 삭제)

### 데이터
- 프로젝트, 시드 자료, 페르소나 설정, 시뮬레이션, 에이전트 반응, 라운드 기록, 보고서

### 인증
- 없음 (팀 내부 사용)

### "진짜 제품" 체크리스트
- [ ] MiroFish 엔진이 실제로 시뮬레이션을 돌림 (목업 데이터 X)
- [ ] 실제 LLM API 연결 (하드코딩된 응답 X)
- [ ] 실제 서버에 배포 (localhost X)
- [ ] 팀원이 URL로 접속해서 광고 대본을 넣고 결과를 받을 수 있음

## Phase 2: A/B 비교 + 대시보드 + 디스코드

### 전제 조건
- Phase 1이 안정적으로 배포된 상태

### 목표
2개 광고안을 비교하고, 결과를 시각적 대시보드로 확인하며, 디스코드에서도 실행 가능한 상태.

### 기능
- [ ] A/B 비교 시뮬레이션 (동일 페르소나에 광고안 2개 동시 실행)
- [ ] A/B 비교 보고서 (어떤 안이 더 효과적인지 비교 분석)
- [ ] 수치 대시보드 (D3.js 기반 차트: 감정 분포, 연령대별 반응, 라운드별 추이)
- [ ] 디스코드 봇 (`/adsim run "광고대본"` 같은 커맨드로 실행, 결과 요약 전송)
- [ ] 시뮬레이션 히스토리 관리 (과거 결과 비교/검색)

### 추가 데이터
- A/B 비교 결과 엔티티 (두 시뮬레이션 간 비교 메타데이터)

### 통합 테스트
- Phase 1 기능이 여전히 정상 동작하는지 확인

## Phase 3: 고도화

### 전제 조건
- Phase 1 + 2가 안정적으로 운영 중

### 목표
AI가 광고 개선안을 제안하고, 경쟁사 비교와 플랫폼별 예측이 가능한 프리미엄 기능.

### 기능
- [ ] 개선 제안 기능 (시뮬레이션 결과 기반 광고 수정안 자동 생성)
- [ ] 경쟁사 광고 비교 분석 (경쟁사 광고 텍스트 입력 → 내 광고와 비교 시뮬레이션)
- [ ] 플랫폼별 예측 (인스타/유튜브/네이버 각각 다른 에이전트 특성 적용)
- [ ] 팀 멤버 권한 관리 및 코멘트 기능
- [ ] 보고서 PDF 내보내기

### 주의사항
- LLM 비용이 크게 증가할 수 있음 (A/B + 플랫폼별 = 시뮬레이션 4~6회)
- 플랫폼별 에이전트 특성 설계에 도메인 리서치 필요
