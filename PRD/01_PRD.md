# AdSim -- PRD (Product Requirements Document)

> 생성일: 2026-04-09
> 생성 도구: Show Me The PRD

## 1. 제품 개요

### 한 줄 요약
MiroFish의 군집 지능 시뮬레이션 엔진을 활용해, 광고 대본과 제품 USP에 대한 가상 소비자 반응을 미리 예측하는 마케팅 시뮬레이션 플랫폼.

### 해결하는 문제
마케팅 팀이 광고를 집행하기 전에 "이 광고가 먹힐까?", "이 USP가 소비자에게 와닿을까?"를 알 방법이 없음. 기존 도구들(Zappi, Kantar 등)은 비싸고(연간 수천만원), 느리고(며칠~주), 한국어 지원이 부족함. 소규모 팀은 감으로 결정하거나, 실제 광고비를 써가며 A/B 테스트할 수밖에 없음.

### 핵심 가치
- **군집 시뮬레이션**: 단순 AI 점수가 아닌, 수십~수백 명의 가상 소비자가 실제로 대화하며 반응하는 "살아있는" 시뮬레이션
- **대화 로그 열람**: 가상 소비자들이 왜 좋다/싫다고 느끼는지 대화를 직접 읽을 수 있음 (블랙박스 아님)
- **한국어 네이티브**: 한국 시장과 한국어 광고에 최적화
- **저렴한 진입**: MiroFish 오픈소스 기반이라 인프라 비용만 발생

## 2. 사용자

### 주요 사용자
- **누구**: 마케팅 팀/광고 기획자/브랜드 매니저 (10~100명 규모의 팀 또는 커뮤니티)
- **상황**: 신규 광고 캠페인 기획 중, 제품 USP 검증이 필요할 때
- **목표**: 광고 집행 전에 타겟 소비자의 반응을 미리 파악하고 싶음

### 사용자 시나리오

**시나리오 1: 광고 대본 테스트**
1. 마케팅 매니저가 새 TV광고 대본을 AdSim에 업로드하고
2. "2030 여성, 건강에 관심 많음" 타겟 페르소나를 설정한 뒤 시뮬레이션을 실행하면
3. 50명의 가상 소비자가 광고에 대해 자유롭게 반응하고, "긍정 72%, 핵심 우려: 가격이 안 보임" 같은 보고서를 받음

**시나리오 2: USP 검증**
1. 제품 매니저가 신제품 USP 3가지를 입력하고
2. "3040 직장인" 타겟을 설정해 시뮬레이션을 돌리면
3. 어떤 USP가 가장 구매 의향을 높이는지, 어떤 건 반응이 없는지 비교 결과를 받음

## 3. 핵심 기능

| 기능 | 설명 | 우선순위 | 복잡도 |
|------|------|----------|--------|
| 시드 자료 업로드 | 광고 대본, USP 텍스트, 경쟁사 정보를 텍스트/PDF로 업로드 | P1 (MVP) | 간단 |
| 타겟 페르소나 설정 | 연령, 성별, 관심사, 소비습관 등으로 가상 소비자 집단 정의 | P1 (MVP) | 간단 |
| 광고 반응 시뮬레이션 | MiroFish 엔진으로 가상 소비자들이 광고에 대해 대화/반응하는 시뮬레이션 실행 | P1 (MVP) | 보통 |
| USP 시장 반응 테스트 | 제품 USP를 입력하면 가상 소비자들이 구매 의향/반응을 시뮬레이션 | P1 (MVP) | 보통 |
| 텍스트 분석 보고서 | 긍정/부정 비율, 핵심 인사이트, 우려 사항 등을 정리한 분석 리포트 | P1 (MVP) | 보통 |
| 에이전트 대화 로그 | 가상 소비자들의 실제 대화 내용을 열람 | P1 (MVP) | 간단 |
| A/B 비교 테스트 | 2개 광고안을 동시에 시뮬레이션해서 비교 | P2 | 복잡 |
| 수치 대시보드 | 차트, 연령대별 반응 그래프 등 시각화 | P2 | 보통 |
| 디스코드 봇 | 팀 채널에서 바로 시뮬레이션 실행/결과 확인 | P2 | 보통 |
| 개선 제안 기능 | 시뮬레이션 결과 기반 광고 수정안 자동 생성 | P3 | 복잡 |

## 4. 사용자 흐름 (User Flow)

### 핵심 흐름
```
시드 자료 업로드 -> 타겟 페르소나 설정 -> 시뮬레이션 실행 -> 보고서 확인 -> 대화 로그 열람
```

### 상세 흐름
1. **시드 자료 업로드**: 광고 대본 텍스트를 붙여넣거나 PDF를 드래그앤드롭
2. **시뮬레이션 유형 선택**: "광고 반응 테스트" 또는 "USP 시장 반응 테스트" 선택
3. **타겟 설정**: 미리 준비된 페르소나 프리셋(2030 여성, 3040 직장인 등) 선택 또는 커스텀 설정
4. **시뮬레이션 실행**: MiroFish 엔진이 에이전트를 생성하고 시뮬레이션 진행 (진행률 표시)
5. **보고서 확인**: 종합 분석 보고서 열람 (긍정/부정 비율, 핵심 인사이트)
6. **대화 로그 탐색**: 개별 가상 소비자의 대화를 직접 읽으며 구체적 인사이트 파악

---

## 5. 성공 기준

- [ ] 광고 대본을 업로드하고 30분 이내에 시뮬레이션 결과를 받을 수 있다
- [ ] 보고서에서 "왜 긍정/부정인지" 이유를 에이전트 대화로 확인할 수 있다
- [ ] 팀원 3명 이상이 동시에 접속해서 각자 프로젝트를 실행할 수 있다
- [ ] USP 2~3개를 입력하면 어떤 것이 가장 반응이 좋은지 순위를 알 수 있다

## 6. 안 만드는 것 (Out of Scope)

> 이 목록에 있는 건 Phase 1에서 만들지 않습니다.
> AI에게 코드를 시킬 때 이 목록을 함께 공유하세요.

- A/B 비교 테스트 -- 이유: Phase 2에서 추가. MVP에서는 단일 시뮬레이션에 집중
- 수치 대시보드/차트 -- 이유: Phase 2에서 D3.js 활용하여 추가
- 디스코드 봇 -- 이유: Phase 2에서 추가. 웹이 먼저 안정화되어야 함
- 개선 제안 기능 -- 이유: Phase 3 고도화 기능
- 경쟁사 광고 크롤링/비교 -- 이유: Phase 3 고도화 기능
- 로그인/회원가입 -- 이유: 팀 내부 사용이므로 불필요
- 결제/과금 시스템 -- 이유: MVP 단계에서 불필요
- 모바일 앱 -- 이유: 광고 기획은 데스크탑에서 하므로 웹만으로 충분

## 7. 확정된 설정

| 항목 | 결정 | 비고 |
|------|------|------|
| 에이전트 수 | **사용자가 매번 선택** (기본값 30명) | 10~100명 범위, 슬라이더로 조절 |
| 시뮬레이션 라운드 | **기본 30라운드** | 사용자가 조절 가능 |

| DB | **SQLite** | 파일 하나로 동작, 나중에 PostgreSQL 이관 가능 |
| 배포 | **로컬/내부 서버 (Docker)** | 무료, 팀 내부 사용 |
